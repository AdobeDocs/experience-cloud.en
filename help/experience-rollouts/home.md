# Adobe Experience Rollouts {#experience-rollouts-home}

Adobe Experience Rollouts is a feature management and controlled release platform that helps product teams deliver new capabilities safely, gradually, and with less risk.

Instead of enabling a new feature for everyone at once, Experience Rollouts lets product teams:

* **Release features gradually** — Start with a small percentage of users, then increase over time. If something does not behave as expected, the feature can be turned off quickly without downtime or reinstalling software.

* **Target features to specific audiences** — Enable new capabilities for selected groups (for example, beta programs, certain regions, or specific plans) so changes can be validated and refined before they are broadly available.

* **Run experiments and A/B tests** — Try different versions of a feature (such as alternate layouts, flows, or recommendations) and measure which works better, using data to guide product decisions.

* **Reduce risk for critical releases** — Break large changes into smaller, controlled rollouts so issues are easier to detect, understand, and fix before they affect everyone.

* **Respond quickly to issues** — If a bug or performance problem appears, turn off only the affected feature while keeping the rest of the application stable.

In practice, Experience Rollouts helps end users experience more frequent improvements with fewer disruptions: new capabilities can arrive sooner, be tested more carefully, and be adjusted quickly based on real-world feedback.
